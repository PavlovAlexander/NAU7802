## 🎯 Суть Проекта в Одной Строке

Создать приложение для тестирования и калибровки цифровых весов на базе **ESP-32 + NAU7802** (24-bit ADC) с тензодатчиком 50g.

## 🚀 Первые Шаги (Фаза 1)

### 1. Создать platformio.ini

```ini
[env:esp32doit-devkit-v1]
platform = espressif32
board = esp32doit-devkit-v1
framework = arduino
monitor_speed = 115200

lib_deps = 
    sparkfun/SparkFun NAU7802 Scale@^1.2.5
```

### 2. Базовый тест

```cpp
// src/main.cpp
#include <Arduino.h>
#include "SparkFun_NAU7802_Scale.h"

SparkFun_NAU7802 myScale;

void setup() {
  Serial.begin(115200);
  Wire.begin(); // GPIO 21 (SDA), GPIO 22 (SCL)
  
  if (!myScale.begin(Wire)) {
    Serial.println("[ERROR] NAU7802 not found!");
    while(1);
  }
  
  myScale.selectGain(64);
  myScale.setSampleRate(320);
  myScale.enableChopper();
  
  Serial.println("[OK] NAU7802 initialized!");
}

void loop() {
  if (myScale.available()) {
    int32_t raw = myScale.getReading();
    Serial.printf("Raw ADC: %ld\n", raw);
  }
  delay(100);
}
```

### 3. Upload и Monitor
```bash
PlatformIO: Upload
PlatformIO: Monitor
```

## 🔬 Калибровка: Формулы

### Линейная регрессия (4 точки)

```
y = k*x + b

где:
  y = реальный вес (0, 10, 20, 30 грамм)
  x = сырое ADC значение от NAU7802
  k = коэффициент наклона (slope)
  b = смещение (intercept)

Расчёт методом наименьших квадратов:
  k = (Σ(xy) - n*mean(x)*mean(y)) / (Σ(x²) - n*mean(x)²)
  b = mean(y) - k*mean(x)

Качество подгонки (R²):
  R² = 1 - (SS_res / SS_tot)
  
  Требуемое: R² > 0.99 (отличная подгонка)
  Допустимое: R² > 0.95 (хорошая подгонка)
  Неприемлемое: R² < 0.95 (повторить калибровку)
```

## 🔌 Аппаратная Конфигурация

### I²C Пины ESP-32
```
ESP-32 GPIO 21 (SDA) ← NAU7802 SDA
ESP-32 GPIO 22 (SCL) ← NAU7802 SCL
ESP-32 GND           ← NAU7802 GND
ESP-32 3V3           ← NAU7802 VCC
```

### NAU7802 конфигурация
```
Адрес I2C:        0x2A
Питание:          3.3V (от ESP-32)
Частота I2C:      400 кГц
Gain:             128x (максимальный для достижения разрешения 1мг)
Sample Rate:      320 SPS (Sample Per Second) для высокого разрешения
Clock Source:     Internal 4.096 MHz
Chopper:          Enabled (для уменьшения шума)
Excitation:       Internal LDO ~10V (питание тензодатчика)
Resolution:       24-bit (достаточно для 1мг разрешения на 50g)
```

## 📞 Контрольные Вопросы

**P:** Почему 4 точки калибровки?  
**A:** 4 точки позволяют построить надёжную линейную модель и вычислить R² для валидации.

**P:** Почему SparkFun, а не Adafruit?  
**A:** Требуется многоточечная калибровка → SparkFun даёт полный контроль, Adafruit только односточечная.

**P:** Как достичь ±2% точности?  
**A:** Калибровка на 4 точках + SMA-5 фильтр + качественные гирьки + спокойная обстановка.

**P:** Что делать если R² < 0.99?  
**A:** Повторить калибровку. Проверить: нет ли вибраций, гирьки правильного веса, стабильность питания.
