# ESP32 NAU7802 Precision Scale Firmware

Прошивка для высокоточных весов на базе ESP32 и 24-битного АЦП NAU7802 с тензодатчиком.

## Характеристики

- **Точность**: ±1-2 мг при 30g (с компонентами точности)
- **Диапазон**: 0-30g
- **Частота измерений**: ~5 Hz (адаптивная)
- **Калибровка**: 4 метода (линейная, кусочно-линейная, полиномиальная 2/3 степени)
- **Интерфейс**: Serial консоль (115200 baud)

## Компоненты точности

### 1. Нелинейная калибровка ✅
- **Линейная регрессия** — базовый метод (обратная совместимость)
- **Кусочно-линейная интерполяция** — 3 сегмента между 4 точками
- **Полиномиальная регрессия** — степени 2 и 3
- Автоматический выбор лучшего метода по R²

### 2. Температурная компенсация ✅
- Использование встроенного температурного сенсора NAU7802
- Коррекция коэффициентов k и b: k' = k × (1 + α × ΔT), b' = b + β × ΔT
- Двухточечная калибровка для вычисления α и β
- Дефолтные коэффициенты: α=20 ppm/°C, β=0.5 мг/°C

### 3. Фильтрация выбросов ✅
- **Медианный фильтр** — окно 3/5/7 отсчётов
- **Статистический 3σ фильтр** — окно 10-20 отсчётов
- Автоматическое отбрасывание аномальных значений
- Статистика: количество отброшенных выбросов

### 4. Адаптивное усреднение ✅
- Автоматический выбор количества отсчётов (10-50)
- Больше отсчётов при стабильном весе → точность
- Меньше отсчётов при изменении веса → скорость
- Плавный переход на основе σ (порог 5 мг)

## Оптимальная конфигурация

- **Sample Rate**: 80 SPS
- **Gain**: 128x
- **I2C Clock**: 100 kHz (стабильность)
- **Chopper**: Отключён (CLK_CHP = 0b11)
- **Усреднение**: Адаптивное (10-50 отсчётов)

## Быстрый старт

### Требования

- PlatformIO
- ESP32 DevKit v1
- NAU7802 breakout board (SparkFun или аналог)
- Тензодатчик (load cell)
- Эталонные гирьки (0g, 10g, 20g, 30g)

### Подключение

```
ESP32          NAU7802
GPIO 21 (SDA) → SDA
GPIO 22 (SCL) → SCL
3.3V          → VCC
GND           → GND
```

### Сборка и загрузка

```bash
pio run -t upload
pio device monitor
```

### Калибровка

1. При первом запуске нажмите ENTER для калибровки
2. Последовательно размещайте гирьки 0g, 10g, 20g, 30g
3. Нажимайте ENTER после каждой гирьки
4. Калибровка сохраняется в NVS автоматически

### Команды консоли

#### Базовые команды
- `C` — Запустить калибровку
- `S` — Показать статус калибровки
- `R` — Перезагрузка системы
- `L` — Загрузить сохранённую калибровку (при старте)
- `N` — Новая калибровка (при старте)

#### Команды управления точностью
- `cmd_accuracy_status()` — Статус всех компонентов
- `cmd_accuracy_enable(component)` — Включить компонент (outlier/temp/adaptive)
- `cmd_accuracy_disable(component)` — Выключить компонент
- `cmd_accuracy_set_outlier_method(method)` — Метод фильтрации (median/sigma/none)
- `cmd_accuracy_set_outlier_window(size)` — Размер окна (3/5/7)
- `cmd_accuracy_set_temp_alpha(value)` — Температурный коэффициент α
- `cmd_accuracy_set_temp_beta(value)` — Температурный коэффициент β
- `cmd_accuracy_set_adaptive_threshold(value)` — Порог стабильности (мг)
- `cmd_accuracy_set_adaptive_samples(min, max)` — Диапазон отсчётов
- `cmd_accuracy_export()` — Экспорт конфигурации
- `cmd_accuracy_import(text)` — Импорт конфигурации
- `cmd_accuracy_reset()` — Сброс к дефолту
- `cmd_accuracy_save()` — Сохранить в NVS

#### Диагностические команды
- `cmd_accuracy_diag_on()` — Включить режим диагностики
- `cmd_accuracy_diag_off()` — Выключить режим диагностики
- `cmd_accuracy_stats()` — Статистика (uptime, выбросы, компенсации)
- `cmd_accuracy_log_export()` — Экспорт лога в CSV
- `cmd_accuracy_log_clear()` — Очистить лог

#### Тестовые команды
- `cmd_accuracy_test_stability(scale, cal)` — Тест стабильности (100 измерений)
- `cmd_accuracy_compare_calibration(scale, cal)` — Сравнение методов калибровки

## Известные проблемы

1. **I2C ошибки (Error 263)**: Возникают при приближении руки к устройству (электромагнитные помехи). Решение: экранирование проводов + фильтр выбросов ✅
2. **Выбросы при изменении веса**: Механические вибрации. Решено медианным/3σ фильтром ✅
3. **Температурный дрейф**: ~0.5-1 мг при ΔT=10°C. Решено температурной компенсацией ✅
