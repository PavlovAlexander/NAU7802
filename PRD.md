# PRD: Тестер Весов ESP-32 с NAU7802

**Версия:** 1.0  
**Дата:** Апрель 2026  
**Статус:** Активная разработка

---

## 1. Обзор Проекта

### 1.1 Описание
Разработка программной части системы тестирования и калибровки цифровых весов на базе:
- **Микроконтроллер:** ESP-32
- **Контроллер взвешивания:** NAU7802 (24-итный ADC с I2C интерфейсом)
- **Датчик:** Тензодатчик 50 грамм (half-bridge)
- **Среда разработки:** PlatformIO

### 1.2 Цель
Создать приложение для тестирования функциональности связки ESP-32 + NAU7802 + тензодатчик с возможностью калибровки и взвешивания в реальном времени.

### 1.3 Ключевые Показатели Успеха
- ✅ Успешная инициализация NAU7802 при старте
- ✅ Калибровка на 4 точках (0, 10, 20, 30 грамм)
- ✅ Точность взвешивания: ±1 миллиграмм (±0.001 грамма) в диапазоне 0-50 грамм
- ✅ Частота считывания: 10 Гц (10 измерений/сек)
- ✅ Время отклика интерфейса: <100 мс
- ✅ Разрешение: 1 миллиграмм (класс точности 0.001% R.O.)

### 2.2 Этап 1: Инициализация (Startup)
**Требования:**
- При включении автоматически инициировать I2C (GPIO 21 - SDA, GPIO 22 - SCL)
- Проверить доступность NAU7802 (адрес 0x00 по умолчанию)
- Запустить "Chopper Stabilization" (встроенная компенсация)
- Установить режим работы: Internal Clock, 320 SPS (Sample Per Second)
- Вывести статус инициализации в UART (115200 baud)

**Вывод в консоль:**
```
[INIT] ESP-32 NAU7802 Scale Tester v1.0
[I2C] Scanning I2C bus...
[NAU7802] Device found at 0x2A
[NAU7802] Revision: 0x0F
[NAU7802] Chopper enabled
[NAU7802] Ready. Press ENTER for calibration mode.
```

### 2.3 Этап 2: Режим калибровки (Calibration Mode)
**Триггер:** Нажатие ENTER в консоли

**15-шаговый процесс калибровки:**
```
[CALIB] Calibration mode started
[CALIB STEP 1/4] Place NO WEIGHT (0g) on scale. Press ENTER to confirm.
  → Collect 50 samples at 10 Hz (5 seconds)
  → Average raw value: R₀
  → Store: calibration[0] = {weight: 0, raw: R₀}

[CALIB STEP 2/4] Place 10g WEIGHT on scale. Press ENTER to confirm.
  → Collect 50 samples at 10 Hz (5 seconds)
  → Average raw value: R₁₀
  → Store: calibration[1] = {weight: 10, raw: R₁₀}

[CALIB STEP 3/4] Place 20g WEIGHT on scale. Press ENTER to confirm.
  → Collect 50 samples at 10 Hz (5 seconds)
  → Average raw value: R₂₀
  → Store: calibration[2] = {weight: 20, raw: R₂₀}

[CALIB STEP 4/4] Place 30g WEIGHT on scale. Press ENTER to confirm.
  → Collect 50 samples at 10 Hz (5 seconds)
  → Average raw value: R₃₀
  → Store: calibration[3] = {weight: 30, raw: R₃₀}
```

**Расчеты калибровки:**
```
1. Построить кривую линейной регрессии через 4 точки:
   y = kx + b
   где x = raw ADC values, y = actual weights

2. Использовать метод наименьших квадратов (Least Squares):
   k = (Σ(xy) - n*mean(x)*mean(y)) / (Σ(x²) - n*mean(x)²)
   b = mean(y) - k*mean(x)

3. Вычислить R² (коэффициент детерминации) для валидации:
   R² = 1 - (SS_res / SS_tot)
   Порог: R² > 0.99

4. Сохранить в EEPROM/NVS:
   - k (коэффициент)
   - b (смещение)
   - R² (качество)
   - Дату калибровки
```

**Вывод результатов:**
```
[CALIB] Calibration complete!
[CALIB] Calibration points:
  0g:  2845012 ADC
  10g: 2865234 ADC
  20g: 2885456 ADC
  30g: 2905678 ADC
[CALIB] Linear fit: y = 0.00000324*x - 8.247
[CALIB] R² = 0.9998 ✓ (Excellent fit)
[CALIB] Data saved to EEPROM
[CALIB] Press ENTER to start weighing...
```

### 2.4 Этап 3: Режим взвешивания (Weighing Mode)
**Триггер:** ENTER после калибровки (или при запуске если калибровка существует)

**Требования:**
- Непрерывное считывание из NAU7802 с частотой 10 Гц
- Обработка: `weight = k * rawValue + b`
- Стабилизация показаний (SMA-5 среднее скользящее за последние 5 измерений)
- Вывод каждого измерения с временной меткой

**Формат вывода:**
```
[WEIGH] Started
[WEIGH] Raw ADC: 2875345 | Calibrated: 12.45g
[WEIGH] Raw ADC: 2875352 | Calibrated: 12.46g
[WEIGH] Raw ADC: 2875348 | Calibrated: 12.45g
  ... (бесконечно, 10 Гц = каждые 100 мс)
```

### 2.5 Команды Пользователя
| Команда | Режим       | Действие                       |
|---------|-------------|--------------------------------|
| ENTER   | Ready       | Войти в режим калибровки       |
| ENTER   | Calibration | Подтвердить текущую точку      |
| ENTER   | Weighing    | Профилактическое переиндексирование |
| 'c'     | Weighing    | Повторная калибровка           |
| 'r'     | Любой       | Перезагрузка ESP-32            |

**Источник:** https://github.com/adafruit/Adafruit_NAU7802

| Критерий | Оценка | Примечание |
|----------|--------|-----------|
| **Полнота API** | ⭐⭐⭐ | Базовая функциональность |
| **Документация** | ⭐⭐⭐ | Стандартная Adafruit документация |

**Основные методы:**
```cpp

enableChopper()                // Стабилизация
```

**Преимущества:**

- ✅ Простой API для быстрого прототипирования

**Недостатки:**
- ❌ Зависит от Adafruit_BusIO (дополнительная библиотека)
- ❌ Калибровка только по одной точке (не многоточечная)
- ❌ Меньше контроля над параметрами NAU7802
- ❌ Меньше сообщества, медленнее обновления

---

### 3.3 РЕКОМЕНДАЦИЯ: **SparkFun NAU7802**

**Решение:** Использовать **SparkFun NAU7802** как основную библиотеку.

**Обоснование:**
1. **Для этого проекта требуется многоточечная калибровка (4 точки)** → Adafruit не подходит (одна точка)
2. **Максимальная гибкость** → SparkFun даёт прямой контроль над регистрами
3. **Меньше зависимостей** → ESP32 вольно работает с Wire.h
4. **Лучшая поддержка сообществом** → При проблемах легче найти решение
5. **Оптимизирована для микроконтроллеров** → Меньше памяти

**Имплементация:**
- Использовать SparkFun для низкоуровневого доступа
- Написать собственный слой калибровки (более мощный, чем встроенная Adafruit)
- Добавить сохранение в NVS (Non-Volatile Storage) ESP32

## 4. Технические Требования

### 4.1 Аппаратное обеспечение
```
ESP-32 DevKit:
  - GPIO 21 (SDA) ← NAU7802 SDA
  - GPIO 22 (SCL) ← NAU7802 SCL
  - GND          ← NAU7802 GND
  - 3V3          ← NAU7802 VCC
  - USB          → UART (logs)

NAU7802:
  - I²C адрес: 0x2A
  - Питание: 3.3V
  - Частота I²C: 400 кГц
  - Интегральный стабилизатор
  - Встроенный Chopper для помех

Тензодатчик (Pressure/Load Cell):
  - Диапазон измерений: 50 грамм
  - Разрешение: 1 миллиграмм (±0.001g)
  - Габаритные размеры: 71.8 x 9 x 5 мм
  - Тип подключения: half-bridge (4-проводная)
  - Чувствительность выхода: (0.45 – 0.55) мВ/В
  - Выходной сигнал в нуле: (±0.150) мВ/В
  - Класс точности: 0.001% R.O. (от полного диапазона)
  - Ползучесть (1 минута): -0.001% R.O.
  - Температурный дрейф нуля: Скомпенсирован
  - Безопасная перегрузка: 150% (75g)
  - Входное сопротивление: 1115 Ом ±5%
  - Выходное сопротивление: 1000 Ом ±5%
  - Рекомендуемое напряжение питания: 10 В (пост. ток)
  - Максимальное напряжение питания: 15 В (пост. ток)
  - Терморезистор компенсации: 115 Ом
  - Питание: Внутреннее NAU7802 excitation (~10V)
```

### 4.2 Настройки NAU7802
```cpp
// Реестры конфигурации
ADC Channel:     CH1 (по умолчанию)
SPS (Sample Rate): 320 SPS (3.125 ms per sample) для максимальной точности
Clock Source:     Internal 4.096 MHz
Gain:             128x (максимальный для достижения разрешения 1мг на 50g)
Excitation:       Internal LDO ~10V (от NAU7802)
Chopper:          Enabled (Anti-aliasing, noise reduction)
Calibration Mode: Internal calibration available
I2C Address:      0x2A
Resolution:       24-bit (достаточно для 1мг разрешения)
```

### 4.3 Параметры Взвешивания
```cpp
// Расчётные пороги для достижения точности ±1мг
SAMPLE_RATE:      10 Hz (0x00 = 320 SPS / 32)
BUFFER_SIZE:      50 samples (5 секунд на 10 Hz для стабильной калибровки)
AVERAGING:        SMA-10 (Simple Moving Average из 10 последних значений)
DEADZONE:         ±0.001g (фильтр шума, 1 миллиграмм)
UPDATE_INTERVAL:  100 ms (1/10 Hz)
DIGITAL_FILTER:   Enabled (встроенный фильтр NAU7802)
NOISE_FLOOR:      <0.5мг (целевой)
```

### 5.2 Состояния FSM (Finite State Machine)

```
State Machine States:
┌─────────────────────────────────────────────┐
│         STATE DIAGRAM                       │
└─────────────────────────────────────────────┘

INIT_HARDWARE
    ↓
    ├─→ [FAIL] → ERROR_HANDLER
    ↓
WAITING_FOR_CALIBRATION (ждём ENTER)
    ↓
CALIBRATION_POINT_1 (0g, сбор 50 samples)
    ↓
CALIBRATION_POINT_2 (10g, сбор 50 samples)
    ↓
CALIBRATION_POINT_3 (20g, сбор 50 samples)
    ↓
CALIBRATION_POINT_4 (30g, сбор 50 samples)
    ↓
CALIBRATION_CALCULATE (расчёт k, b, R²)
    ├─→ [FAIL: R² < 0.99] → RECALIBRATION_REQUIRED
    ↓
CALIBRATION_SAVE (сохранить в NVS)
    ↓
WEIGHING_MODE (бесконечный цикл at 10 Hz)
    ├─→ ['c'] → Переход в CALIBRATION_POINT_1
    ├─→ ['r'] → REBOOT
    └─→ ['s'] → Print diagnostic status
```

## Приложение: Справочные Материалы

### A. NAU7802 Datasheet specs
- 24-bit ADC with ±16-bit input range
- I²C Interface (0x2A адрес поддерживается через конфигурацию)
- Power supply: 2.7V - 5.5V (используется 3.3V от ESP-32)
- Current draw: 1.3 mA (typical)
- Chopper: Schmit trigger stabilization (включена по умолчанию)
- Excitation: Internal 10V LDO (для питания тензодатчика)
- Resolution: 24-bit (достаточно для разрешения 1мг на 50g датчике)
- Gain: программируемый от 1x до 128x
